# Testing and 

Testing is a **systematic process** of **evaluating** a product, system, or component to **detect differences** between expected and actual results, **verify** compliance with specified requirements and **identify potential defects** or areas for **improvement**.

Here are some kinds of tests:

* Unit testing: test individual components of functions in isolation
* Integration testing: verifies that different modules or services work together correctly
* Functional testing: ensure that the software meets specified functional requirements
* End-to-End (E2E) testing: test the entire application flow from start to finish
* Acceptance testing: test which determines if the system satisfies the acceptance criteria and is ready for production
* Performance testing: evaluates the system’s performance under various conditions
* Load testing: assesses how the system performs under expected load conditions
* Stress testing: examines system behavior under extreme conditions or beyond normal operational capacity
* Security testing: identifies vulnerabilities and ensures the system is protected against various types of attacks
* Usability testing: evaluates the user interface and overall user experience
* Smoke testing: quick test to check if the basic functionality works before more thorough testing

---

## Static vs Dynamic Testing

However we distinguish between two main categories of tests:

| **Static**                                                                                                      	| **Dynamic**                                                   	|
|-----------------------------------------------------------------------------------------------------------------	|---------------------------------------------------------------	|
| Performed **without executing** the code                                                                        	| Involves **executing** the code                               	|
| Involves reviewing **documents, code**, and other artifacts                                                     	| Observes the **behavior** of the (running) system under test  	|
| Can be done **early** in the development process. Therefore, it can catch issues early and improve code quality 	| Usually performed **later** in the development process        	|
| Examples: code reviews, static code analysis, design and architecture reviews, documentation review             	| Examples: unit, integration, system, and performance testings 	|
